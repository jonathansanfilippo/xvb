Qu’est-ce que XVB.

XVB (Extended Video Broadcasting) est un cœur de lecture web entièrement client-side. Il est conçu pour gérer des sources IPTV et “broadcast style” directement dans le navigateur, avec un focus sur la fiabilité, la compatibilité et les performances.

Ce projet n’est pas “une plateforme IPTV backend” — c’est un moteur de lecteur. Vous fournissez les sources (playlists, flux, fichiers) ; XVB gère la lecture.

Pourquoi c’est important.

  • Lecture stable sur de longues durées (live ou VOD)
  • Démarrage rapide et zapping fluide
  • Compatible avec les formats IPTV courants
  • Séparation claire entre moteur et interface

Support streaming.

Conçu pour l’IPTV et une lecture live résiliente sur réseaux instables.

Protocoles & playlists

XVB prend en charge plusieurs technologies de streaming utilisées en IPTV et diffusion :

  • HLS (HTTP Live Streaming, .m3u8)
  • MPEG-DASH (.mpd)
  • MPEG-TS flux live (.ts)
  • M3U / M3U8 playlists
  • Flux MP4 standard

Formats lisibles

Selon le navigateur et la configuration du flux, XVB peut lire :

  • Vidéo : m3u8, mp4, webm, ogv, ts
  • Audio : mp3, aac, m4a, ogg, opus, wav, flac

Les codecs disponibles varient selon les navigateurs, appareils et systèmes.

Guide utilisateur

Comprendre l’interface

XVB garde la barre supérieure minimale : indicateurs essentiels + recherche, et un seul raccourci pour gérer les playlists.

Statut en direct

  • Chaînes : Indique combien de flux sont chargés depuis vos playlists actives.
  • Heure EPG : Horodatage de la dernière mise à jour du guide (EPG).
  • En ligne : Nombre d’utilisateurs actuellement connectés à l’écosystème XVB.

Recherche

  • Filtre instantané : Tapez pour filtrer les chaînes en temps réel — sans défilement.
  • Astuce : Utilisez des mots-clés courts (ex. “BBC”, “Sport”, “News”) pour aller plus vite.

Gestion des playlists

  • Ajouter des sources : Enregistrez des URL M3U, importez des fichiers locaux ou téléchargez des listes officielles.
  • Supprimer des éléments : Supprimez des playlists individuellement (sans tout effacer).

Note : Les changements effectués dans le Playlist Manager sont stockés localement dans votre navigateur et appliqués immédiatement au lecteur.

Playlists

Créez votre playlist M3U.

XVB peut charger des playlists via URL ou fichier local. Le format le plus courant est M3U / M3U8 : chaque chaîne a des métadonnées (nom, groupe, logo, id EPG) suivies de l’URL du flux.

Bonnes pratiques : utilisez group-title (catégories), ajoutez un tvg-logo et définissez tvg-id si vous avez un EPG.

Exemple M3U

Enregistrez en my-playlist.m3u (UTF-8). Puis chargez dans XVB. 

#EXTM3U

#EXTINF:-1 tvg-name="Demo Channel 1" tvg-logo="https://example.com/logo1.png" tvg-id="demo.channel.1" group-title="News",Demo Channel 1
https://example.com/stream/channel1.m3u8

#EXTINF:-1 tvg-name="Demo Radio" tvg-logo="https://example.com/radio.png" tvg-id="demo.radio" group-title="Radio",Demo Radio
https://example.com/audio/radio.mp3

#EXTINF:-1 tvg-name="Demo VOD" tvg-logo="https://example.com/vod.png" group-title="On Demand",Demo VOD
https://example.com/video/demo.mp4

Astuce : l’URL peut pointer vers m3u8, mp4, ts, ou audio mp3/aac/m4a.

Comment charger des playlists dans XVB

  • URL M3U : ajoutez une URL de playlist et rafraîchissez les chaînes.
  • Fichier local : chargez un fichier .m3u / .m3u8 depuis votre appareil.
  • Playlist Manager : gérez les playlists individuellement depuis manager.html.
  • Catégories : utilisez group-title pour organiser la barre latérale.
  • EPG : définissez tvg-id pour faire correspondre les guides (si disponibles).

Playlists

Gestion des playlists.

Gérez vos playlists individuellement via une page dédiée. Ajoutez des URL, importez des fichiers M3U, téléchargez des listes officielles et supprimez des éléments un par un — sans tout effacer.

Fonctionnalités

  • Ajouter une URL : enregistrez un lien M3U/M3U8 comme élément unique.
  • Ajouter un fichier : importez un .m3u/.m3u8 et stockez-le dans votre navigateur.
  • Listes serveur : téléchargez des playlists fournies par XVB en un clic.
  • Suppression individuelle : supprimez une playlist sans toucher aux autres.
Ouvrir le Playlist Manager

Comment ça marche

Le manager stocke les playlists dans votre navigateur via LocalStorage. Aucun compte, aucune base de données serveur. Vos listes restent après actualisation et s’appliquent directement au lecteur.

Astuce : lorsque le lecteur est ouvert, les changements se synchronisent instantanément via BroadcastChannel.

Notes & limites

  • Même navigateur / même appareil : les playlists sont stockées localement sur votre appareil.
  • Règles CORS : les playlists distantes doivent autoriser les requêtes cross-origin.
  • HTTPS recommandé : évitez le mixed content (page HTTPS + flux HTTP).

Notes techniques

Limitations navigateur & playlists.

Comme XVB fonctionne entièrement dans le navigateur, le chargement des playlists et la lecture dépendent des règles de sécurité du web. Voici les limites les plus courantes.

CORS (Cross-Origin Resource Sharing)

Si une playlist ou un flux est hébergé sur un autre domaine, le serveur doit autoriser les requêtes cross-origin. Sans en-têtes CORS (ex. Access-Control-Allow-Origin), le navigateur bloquera le flux.

C’est une restriction de sécurité du navigateur — pas une limite de XVB.

HTTP vs HTTPS (contenu mixte)

Si XVB est chargé en HTTPS, vous ne pouvez pas charger des flux en HTTP. Les navigateurs bloquent le contenu mixte.

Solution : servez playlists et flux en HTTPS avec certificat valide.

Certificats invalides ou auto-signés

Les flux avec certificats SSL expirés, invalides ou auto-signés peuvent échouer.

Compatibilité codecs & navigateur

Tous les codecs ne sont pas supportés partout. Par exemple :

  • H.264 + AAC est largement supporté
  • Certains flux MPEG-TS peuvent échouer selon codec/container
  • Support HEVC / H.265 variable selon navigateur et OS

Authentification & en-têtes

Certains fournisseurs IPTV exigent des en-têtes HTTP personnalisés, tokens ou cookies. Si l’authentification n’est pas accessible via le navigateur, la lecture peut échouer.

Playlists très volumineuses

Les playlists M3U très grandes (milliers de chaînes) peuvent ralentir le chargement initial, selon la mémoire et le CPU.

XVB ne restreint pas le contenu — mais les règles du navigateur, la configuration serveur et la qualité du flux impactent la lecture.

Légal & usage responsable

Utilisez XVB uniquement avec des sources autorisées.

XVB est un outil neutre : il ne fournit ni chaînes, ni identifiants, ni contenu. Vous êtes responsable des sources chargées. Utilisez XVB uniquement avec des flux et playlists auxquels vous êtes autorisé (démos publiques, médias personnels, fournisseurs IPTV sous licence, contenu que vous avez le droit de distribuer).

Évitez la piraterie, les identifiants volés, les services “free IPTV” ou toute méthode visant à contourner paywalls, géo-restrictions ou protections copyright. En cas de doute, n’utilisez pas la source.

Architecture du cœur.

Entièrement client-side. Aucun backend requis pour la lecture. Basé sur des moteurs éprouvés et les API natives du navigateur.
