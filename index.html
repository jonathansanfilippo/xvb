What XVB is.

XVB (Extended Video Broadcasting) is a web-based playback core that runs fully client-side. It’s designed to handle IPTV and broadcast-style sources directly in the browser, with emphasis on reliability, compatibility, and performance.

This project is not “a backend IPTV platform” — it’s a focused player engine. You bring your sources (playlists, streams, or files); XVB handles playback.

Why it matters.

  • Stable long-duration playback (live or VOD)
  • Fast stream initialization and switching
  • Works with common IPTV playlist formats
  • Clean separation between playback engine and UI

Streaming support.

Built for IPTV environments and resilient live playback over unstable networks.

Protocols & playlists

XVB supports multiple streaming technologies commonly used in IPTV and broadcasting:

  • HLS (HTTP Live Streaming, .m3u8)
  • MPEG-DASH (.mpd)
  • MPEG-TS live streams (.ts)
  • M3U / M3U8 playlists
  • Standard MP4 streams

Playable media formats

Depending on browser support and stream configuration, XVB can play the following formats:

  • Video: m3u8, mp4, webm, ogv, ts
  • Audio: mp3, aac, m4a, ogg, opus, wav, flac

Audio and codec availability may vary across browsers, devices, and operating systems.

User Manual

Understanding the Interface

XVB features a minimalist yet powerful toolbar designed for speed and real-time monitoring. Here is a breakdown of every element in your dashboard.

Live Metrics

  • Channels Counter: Displays the total number of playable streams currently loaded from your active playlists.
  • EPG Status: Real-time timestamp of the last Electronic Program Guide update (Italy & UK sources).
  • Online Users: Shows how many users are currently connected to the XVB ecosystem worldwide.

Navigation

  • Smart Search: Instant filtering. Start typing a channel name to find it immediately without scrolling.
  • Sample Cloud: If no lists are present, click this to load a technical sample list to test MP4, AAC, MP3, OGG, MPD, TS, and HLS.

Playlist Management

  • Add Remote URL: Import direct M3U links. Note: This method targets static direct links for maximum performance.
  • Local Import: Browse your device to upload .m3u or .m3u8 files stored locally.
  • Clear Cache: Instantly wipes all stored playlists and data from your browser's local cache.

Pro Tip: Use the Clear Cache button if you experience loading glitches after updating a local file. XVB stores data client-side for ultra-fast startup.

Playlists

Create your own M3U playlist.

XVB can load playlists from an URL or from a local file. The most common format is M3U / M3U8, where each channel has metadata (name, group, logo, EPG id) followed by the stream URL.

Best practice: keep your playlist tidy with group-title (categories), provide a tvg-logo for a clean UI, and set tvg-id when you have an EPG.

M3U example

Save as my-playlist.m3u (UTF-8). Then load it in XVB. 

#EXTM3U

#EXTINF:-1 tvg-name="Demo Channel 1" tvg-logo="https://example.com/logo1.png" tvg-id="demo.channel.1" group-title="News",Demo Channel 1
https://example.com/stream/channel1.m3u8

#EXTINF:-1 tvg-name="Demo Radio" tvg-logo="https://example.com/radio.png" tvg-id="demo.radio" group-title="Radio",Demo Radio
https://example.com/audio/radio.mp3

#EXTINF:-1 tvg-name="Demo VOD" tvg-logo="https://example.com/vod.png" group-title="On Demand",Demo VOD
https://example.com/video/demo.mp4

Tip: the URL can point to m3u8, mp4, ts, or audio like mp3/aac/m4a depending on your sources.

How to load playlists in XVB

  • M3U URL: add a playlist URL and refresh channels.
  • Local file: load an .m3u / .m3u8 file directly from your device.
  • Categories: use group-title to organize channels in the sidebar.
  • EPG: set tvg-id to match guide sources (when available).

Technical notes

Browser & playlist limitations.

Because XVB runs entirely in the browser, playlist loading and playback are subject to standard web security rules. Below are the most common limitations you may encounter.

CORS (Cross-Origin Resource Sharing)

If a playlist or stream is hosted on a different domain, the server must allow cross-origin requests. Without proper CORS headers (for example Access-Control-Allow-Origin), the browser will block the stream.

This is a browser security restriction — not a limitation of XVB.

HTTP vs HTTPS (Mixed Content)

If XVB is loaded over HTTPS, you cannot load streams over plain HTTP. Browsers block mixed content for security reasons.

Solution: ensure your playlist and streams are served over HTTPS with a valid certificate.

Invalid or self-signed certificates

Streams hosted with expired, invalid, or self-signed SSL certificates may fail to load in modern browsers.

Codec & browser compatibility

Not all codecs are supported by all browsers. For example:

  • H.264 + AAC is widely supported
  • Some MPEG-TS streams may fail depending on codec/container
  • HEVC / H.265 support varies by browser and OS

Authentication & headers

Some IPTV providers require custom HTTP headers, tokens, or cookies. If a stream depends on proprietary authentication methods that cannot be accessed by the browser, playback may fail.

Large playlists

Extremely large M3U playlists (thousands of channels) may impact initial loading performance depending on device memory and CPU.

XVB itself does not restrict content — but browser security rules, server configuration, and stream quality directly affect playback behavior.

Legal & responsible use

Use XVB with licensed sources only.

XVB is a neutral playback tool: it does not provide channels, credentials, or content. You are responsible for the sources you load. Use XVB only with streams and playlists you are authorized to access (public demo streams, your own media, licensed IPTV providers, or content you have rights to distribute).

Avoid piracy, stolen credentials, “free IPTV” services, or any method intended to bypass paywalls, geo-restrictions, or copyright protections. If you’re unsure whether a source is legal, don’t use it.
