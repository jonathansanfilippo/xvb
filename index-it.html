Cos’è XVB.

XVB (Extended Video Broadcasting) è un core di riproduzione web che gira interamente client-side. È progettato per gestire sorgenti IPTV e “broadcast style” direttamente nel browser, con enfasi su affidabilità, compatibilità e performance.

Questo progetto non è “una piattaforma IPTV backend” — è un player engine focalizzato. Tu porti le sorgenti (playlist, stream o file); XVB gestisce la riproduzione.

Perché conta.

  • Riproduzione stabile anche per sessioni lunghe (live o VOD)
  • Avvio e cambio canale veloci
  • Compatibile con i formati playlist IPTV più comuni
  • Separazione pulita tra engine e UI

Supporto streaming.

Pensato per ambienti IPTV e riproduzione live resiliente su reti instabili.

Protocolli & playlist

XVB supporta diverse tecnologie di streaming comuni in IPTV e broadcasting:

  • HLS (HTTP Live Streaming, .m3u8)
  • MPEG-DASH (.mpd)
  • MPEG-TS stream live (.ts)
  • M3U / M3U8 playlist
  • Stream MP4 standard

Formati riproducibili

In base al browser e alla configurazione dello stream, XVB può riprodurre:

  • Video: m3u8, mp4, webm, ogv, ts
  • Audio: mp3, aac, m4a, ogg, opus, wav, flac

La disponibilità di codec/audio varia tra browser, dispositivi e sistemi operativi.

Playlist

Crea la tua playlist M3U.

XVB può caricare playlist da URL o da file locale. Il formato più comune è M3U / M3U8, dove ogni canale ha metadati (nome, gruppo, logo, id EPG) seguiti dalla URL dello stream.

Best practice: organizza con group-title (categorie), inserisci un tvg-logo per una UI pulita e imposta tvg-id se hai un EPG.

Esempio M3U

Salva come my-playlist.m3u (UTF-8). Poi caricala in XVB. 

#EXTM3U

#EXTINF:-1 tvg-name="Demo Channel 1" tvg-logo="https://example.com/logo1.png" tvg-id="demo.channel.1" group-title="News",Demo Channel 1
https://example.com/stream/channel1.m3u8

#EXTINF:-1 tvg-name="Demo Radio" tvg-logo="https://example.com/radio.png" tvg-id="demo.radio" group-title="Radio",Demo Radio
https://example.com/audio/radio.mp3

#EXTINF:-1 tvg-name="Demo VOD" tvg-logo="https://example.com/vod.png" group-title="On Demand",Demo VOD
https://example.com/video/demo.mp4

Tip: la URL può puntare a m3u8, mp4, ts, o audio mp3/aac/m4a in base alle sorgenti.

Come caricare playlist in XVB

  • M3U URL: aggiungi una URL playlist e aggiorna i canali.
  • File locale: carica un file .m3u / .m3u8 dal dispositivo.
  • Gestione Playlist: gestisci le playlist singolarmente da manager.html.
  • Categorie: usa group-title per organizzare la sidebar.
  • EPG: imposta tvg-id per matchare le guide disponibili.

Playlist

Gestione Playlist.

Gestisci le playlist singolarmente in una pagina dedicata. Aggiungi URL, importa file M3U locali, scarica liste ufficiali e cancella gli elementi uno per uno — senza azzerare tutto.

Cosa fa

  • Aggiungi URL: salva un link M3U/M3U8 come singolo elemento.
  • Aggiungi file: importa un file .m3u/.m3u8 e salvalo nel browser.
  • Liste server: scarica playlist fornite da XVB con un click.
  • Elimina singolarmente: rimuovi una playlist senza toccare le altre.
Apri Playlist Manager

Come funziona

Il manager salva le playlist nel browser usando LocalStorage. Nessun account, nessun database server. Le liste restano dopo il refresh e si applicano direttamente al player.

Tip: quando il player è aperto, le modifiche si sincronizzano subito tramite BroadcastChannel.

Note e limiti

  • Stesso browser / stesso dispositivo: le playlist sono salvate localmente sul tuo device.
  • Regole CORS: le playlist remote devono consentire richieste cross-origin.
  • HTTPS consigliato: evita blocchi mixed-content (pagina HTTPS + stream HTTP).

Note tecniche

Limiti browser & playlist.

Poiché XVB gira interamente nel browser, caricamento playlist e playback sono soggetti alle regole di sicurezza del web. Qui sotto i limiti più comuni.

CORS (Cross-Origin Resource Sharing)

Se playlist o stream sono su un dominio diverso, il server deve consentire richieste cross-origin. Senza header CORS (es. Access-Control-Allow-Origin), il browser blocca la richiesta.

È una regola di sicurezza del browser — non un limite di XVB.

HTTP vs HTTPS (contenuti misti)

Se XVB è caricato in HTTPS, non puoi caricare stream in HTTP. I browser bloccano i contenuti misti.

Soluzione: usa playlist e stream in HTTPS con certificato valido.

Certificati non validi o self-signed

Stream con certificati scaduti, invalidi o self-signed possono fallire nei browser moderni.

Compatibilità codec & browser

Non tutti i codec sono supportati ovunque. Ad esempio:

  • H.264 + AAC è ampiamente supportato
  • Alcuni MPEG-TS possono fallire a seconda di codec/container
  • Supporto HEVC / H.265 varia per browser e OS

Autenticazione & header

Alcuni provider IPTV richiedono header HTTP custom, token o cookie. Se uno stream dipende da autenticazioni proprietarie non accessibili dal browser, può fallire.

Playlist enormi

Playlist M3U molto grandi (migliaia di canali) possono rallentare il primo caricamento, in base a memoria e CPU del dispositivo.

XVB non limita i contenuti — ma regole browser, configurazione server e qualità stream influenzano il playback.

Uso legale & responsabile

Usa XVB solo con sorgenti autorizzate.

XVB è uno strumento neutro: non fornisce canali, credenziali o contenuti. Sei responsabile delle sorgenti caricate. Usa XVB solo con stream e playlist che sei autorizzato ad usare (demo pubbliche, media personali, provider IPTV con licenza o contenuti che hai diritto di distribuire).

Evita pirateria, credenziali rubate, “free IPTV” o metodi per bypassare paywall, geo-restrizioni o protezioni copyright. Se non sei sicuro della legalità di una sorgente, non usarla.

Architettura core.

Interamente client-side. Nessun backend necessario per il playback. Basato su engine streaming collaudati e API native del browser.
