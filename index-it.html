Cos'è XVB.

XVB (Extended Video Broadcasting) è un core di riproduzione web che gira interamente lato client. È progettato per gestire IPTV e sorgenti broadcast direttamente nel browser, con enfasi su affidabilità, compatibilità e prestazioni.

Questo progetto non è una "piattaforma backend IPTV" — è un motore di riproduzione focalizzato. Tu porti le tue sorgenti (playlist, stream o file); XVB gestisce la riproduzione.

Perché è importante.

  • Riproduzione stabile a lungo termine (live o VOD)
  • Inizializzazione e commutazione rapida dello stream
  • Funziona con i comuni formati di playlist IPTV
  • Netta separazione tra motore di riproduzione e UI

Supporto streaming.

Costruito per ambienti IPTV e riproduzione live resiliente su reti instabili.

Protocolli e playlist

XVB supporta molteplici tecnologie di streaming comunemente usate in IPTV e broadcast:

  • HLS (HTTP Live Streaming, .m3u8)
  • MPEG-DASH (.mpd)
  • MPEG-TS live streams (.ts)
  • M3U / M3U8 playlist
  • Stream standard MP4

Formati media riproducibili

A seconda del supporto del browser e della configurazione, XVB può riprodurre i seguenti formati:

  • Video: m3u8, mp4, webm, ogv, ts
  • Audio: mp3, aac, m4a, ogg, opus, wav, flac

La disponibilità di audio e codec può variare a seconda del browser, del dispositivo e del sistema operativo.

Manuale Utente

Comprendere l'Interfaccia

XVB dispone di una barra degli strumenti minimalista ma potente, progettata per la velocità e il monitoraggio in tempo reale. Ecco un dettaglio di ogni elemento della tua dashboard.

Metriche Live

  • Contatore Canali: Mostra il numero totale di stream riproducibili attualmente caricati dalle tue playlist attive.
  • Stato EPG: Timestamp in tempo reale dell'ultimo aggiornamento della Guida Elettronica ai Programmi (fonti Italia e UK).
  • Utenti Online: Indica quanti utenti sono attualmente connessi all'ecosistema XVB in tutto il mondo.

Navigazione

  • Ricerca Rapida: Filtro istantaneo. Inizia a digitare il nome di un canale per trovarlo immediatamente senza scorrere.
  • Cloud Samples: Se non ci sono liste, clicca qui per caricare una lista di test tecnica per provare MP4, AAC, MP3, OGG, MPD, TS e HLS.

Gestione Playlist

  • Aggiungi URL Remoto: Importa link M3U diretti. Nota: questo metodo punta a link statici diretti per massime prestazioni.
  • Importazione Locale: Sfoglia il tuo dispositivo per caricare file .m3u o .m3u8 salvati localmente.
  • Svuota Cache: Cancella istantaneamente tutte le playlist memorizzate e i dati dalla cache locale del browser.

Consiglio: Usa il tasto Svuota Cache se riscontri problemi dopo l'aggiornamento di un file locale. XVB salva i dati lato client per un avvio ultra-rapido.

Playlist

Crea la tua playlist M3U.

XVB può caricare playlist da un URL o da un file locale. Il formato più comune è M3U / M3U8, dove ogni canale ha metadati (nome, gruppo, logo, EPG id) seguiti dall'URL dello stream.

Suggerimento: mantieni la playlist ordinata con group-title (categorie), fornisci un tvg-logo per una UI pulita e imposta tvg-id se hai un EPG.

Esempio M3U

Salva come mia-playlist.m3u (UTF-8). Poi caricalo in XVB. 

#EXTM3U

#EXTINF:-1 tvg-name="Canale Demo 1" tvg-logo="https://example.com/logo1.png" tvg-id="demo.channel.1" group-title="News",Canale Demo 1
https://example.com/stream/channel1.m3u8

#EXTINF:-1 tvg-name="Radio Demo" tvg-logo="https://example.com/radio.png" tvg-id="demo.radio" group-title="Radio",Radio Demo
https://example.com/audio/radio.mp3

#EXTINF:-1 tvg-name="VOD Demo" tvg-logo="https://example.com/vod.png" group-title="On Demand",VOD Demo
https://example.com/video/demo.mp4

Tip: l'URL può puntare a m3u8, mp4, ts, o audio come mp3/aac/m4a a seconda delle sorgenti.

Come caricare playlist in XVB

  • URL M3U: aggiungi un URL playlist e aggiorna i canali.
  • File locale: carica un file .m3u / .m3u8 direttamente dal dispositivo.
  • Categorie: usa group-title per organizzare i canali nella barra laterale.
  • EPG: imposta tvg-id per abbinare le fonti della guida (quando disponibili).

Note tecniche

Limitazioni del browser e delle playlist.

Poiché XVB gira interamente nel browser, il caricamento e la riproduzione sono soggetti alle regole di sicurezza web standard. Di seguito le limitazioni più comuni.

CORS (Cross-Origin Resource Sharing)

Se una playlist o uno stream è ospitato su un dominio diverso, il server deve consentire le richieste cross-origin. Senza gli header CORS corretti (es. Access-Control-Allow-Origin), il browser bloccherà lo stream.

Questa è una restrizione di sicurezza del browser — non un limite di XVB.

HTTP vs HTTPS (Contenuti Misti)

Se XVB viene caricato via HTTPS, non puoi caricare stream in semplice HTTP. I browser bloccano i contenuti misti per sicurezza.

Soluzione: assicuratevi che playlist e stream siano serviti via HTTPS con certificato valido.

Certificati non validi o self-signed

Gli stream con certificati SSL scaduti, non validi o auto-firmati potrebbero non caricarsi nei browser moderni.

Compatibilità Codec e Browser

Non tutti i codec sono supportati da tutti i browser. Per esempio:

  • H.264 + AAC è ampiamente supportato
  • Alcuni stream MPEG-TS potrebbero fallire a seconda del codec/container
  • Il supporto HEVC / H.265 varia a seconda del browser e dell'OS

Autenticazione e Header

Alcuni fornitori IPTV richiedono header HTTP personalizzati, token o cookie. Se uno stream dipende da metodi di autenticazione proprietari non accessibili dal browser, la riproduzione potrebbe fallire.

Playlist di grandi dimensioni

Playlist M3U estremamente grandi (migliaia di canali) possono influire sulle prestazioni di caricamento iniziale a seconda della memoria e CPU del dispositivo.

XVB di per sé non limita i contenuti — ma le regole del browser, la configurazione del server e la qualità dello stream influenzano direttamente la riproduzione.

Uso legale e responsabile

Usa XVB solo con sorgenti autorizzate.

XVB è uno strumento di riproduzione neutro: non fornisce canali, credenziali o contenuti. Sei responsabile delle sorgenti che carichi. Usa XVB solo con stream e playlist che sei autorizzato ad accedere (demo pubbliche, media personali, fornitori IPTV con licenza).

Evita la pirateria, credenziali rubate o servizi "IPTV gratuiti" destinati a bypassare paywall o protezioni del copyright. Se non sei sicuro della legalità di una sorgente, non usarla.
