Qué es XVB.

XVB (Extended Video Broadcasting) es un núcleo de reproducción web totalmente client-side. Está diseñado para manejar fuentes IPTV y estilo broadcast directamente en el navegador, con énfasis en fiabilidad, compatibilidad y rendimiento.

Este proyecto no es “una plataforma IPTV backend” — es un motor de reproducción. Tú aportas las fuentes (playlists, streams o archivos); XVB gestiona la reproducción.

Por qué importa.

  • Reproducción estable de larga duración (live o VOD)
  • Inicio rápido y cambio de canal ágil
  • Compatible con formatos IPTV comunes
  • Separación clara entre motor y UI

Soporte de streaming.

Diseñado para IPTV y reproducción en vivo resistente en redes inestables.

Protocolos & playlists

XVB soporta varias tecnologías de streaming usadas en IPTV y broadcasting:

  • HLS (HTTP Live Streaming, .m3u8)
  • MPEG-DASH (.mpd)
  • MPEG-TS streams en vivo (.ts)
  • M3U / M3U8 playlists
  • Streams MP4 estándar

Formatos reproducibles

Dependiendo del navegador y la configuración del stream, XVB puede reproducir:

  • Video: m3u8, mp4, webm, ogv, ts
  • Audio: mp3, aac, m4a, ogg, opus, wav, flac

La disponibilidad de codecs puede variar según navegador, dispositivo y sistema.

Playlists

Crea tu playlist M3U.

XVB puede cargar playlists desde una URL o desde un archivo local. El formato más común es M3U / M3U8, donde cada canal tiene metadatos (nombre, grupo, logo, id EPG) seguidos por la URL del stream.

Buenas prácticas: usa group-title (categorías), añade tvg-logo y define tvg-id si tienes EPG.

Ejemplo M3U

Guarda como my-playlist.m3u (UTF-8). Luego cárgala en XVB. 

#EXTM3U

#EXTINF:-1 tvg-name="Demo Channel 1" tvg-logo="https://example.com/logo1.png" tvg-id="demo.channel.1" group-title="News",Demo Channel 1
https://example.com/stream/channel1.m3u8

#EXTINF:-1 tvg-name="Demo Radio" tvg-logo="https://example.com/radio.png" tvg-id="demo.radio" group-title="Radio",Demo Radio
https://example.com/audio/radio.mp3

#EXTINF:-1 tvg-name="Demo VOD" tvg-logo="https://example.com/vod.png" group-title="On Demand",Demo VOD
https://example.com/video/demo.mp4

Tip: la URL puede apuntar a m3u8, mp4, ts, o audio mp3/aac/m4a.

Cómo cargar playlists en XVB

  • URL M3U: añade una URL de playlist y actualiza los canales.
  • Archivo local: carga un archivo .m3u / .m3u8 desde tu dispositivo.
  • Playlist Manager: gestiona playlists individualmente desde manager.html.
  • Categorías: usa group-title para organizar la barra lateral.
  • EPG: define tvg-id para coincidir con guías (si están disponibles).

Playlists

Gestor de playlists.

Gestiona playlists individualmente en una página dedicada. Añade URLs, importa archivos M3U locales, descarga listas oficiales y elimina elementos uno a uno — sin reiniciar todo.

Qué hace

  • Añadir URL: guarda un enlace M3U/M3U8 como elemento único.
  • Añadir archivo: importa un .m3u/.m3u8 y guárdalo en el navegador.
  • Listas del servidor: descarga playlists proporcionadas por XVB con un clic.
  • Borrado individual: elimina una playlist sin tocar las demás.
Abrir Playlist Manager

Cómo funciona

El gestor guarda playlists en tu navegador usando LocalStorage. Sin cuentas, sin base de datos en servidor. Las listas permanecen tras recargar y se aplican directamente al reproductor.

Consejo: con el reproductor abierto, los cambios se sincronizan al instante mediante BroadcastChannel.

Notas y límites

  • Mismo navegador / mismo dispositivo: las playlists se guardan localmente en tu dispositivo.
  • Reglas CORS: las playlists remotas deben permitir solicitudes cross-origin.
  • HTTPS recomendado: evita contenido mixto (página HTTPS + streams HTTP).

Notas técnicas

Limitaciones del navegador y playlists.

Como XVB funciona totalmente en el navegador, la carga de playlists y la reproducción están sujetas a las reglas de seguridad web. Estas son las limitaciones más comunes.

CORS (Cross-Origin Resource Sharing)

Si una playlist o stream está alojado en otro dominio, el servidor debe permitir solicitudes cross-origin. Sin cabeceras CORS (por ejemplo Access-Control-Allow-Origin), el navegador bloqueará el stream.

Es una restricción del navegador — no una limitación de XVB.

HTTP vs HTTPS (contenido mixto)

Si XVB se carga en HTTPS, no puedes cargar streams en HTTP. Los navegadores bloquean el contenido mixto.

Solución: sirve playlists y streams en HTTPS con certificado válido.

Certificados no válidos o autofirmados

Streams con certificados SSL caducados, inválidos o autofirmados pueden fallar.

Compatibilidad de codecs

No todos los codecs están disponibles en todos los navegadores. Por ejemplo:

  • H.264 + AAC está ampliamente soportado
  • Algunos MPEG-TS pueden fallar según codec/contenedor
  • El soporte HEVC / H.265 varía según navegador y sistema

Autenticación y cabeceras

Algunos proveedores IPTV requieren cabeceras HTTP personalizadas, tokens o cookies. Si un stream depende de autenticación propietaria no accesible desde el navegador, la reproducción puede fallar.

Playlists grandes

Playlists M3U muy grandes (miles de canales) pueden afectar el rendimiento de carga inicial, dependiendo de la memoria y CPU del dispositivo.

XVB no restringe contenido — pero reglas del navegador, configuración del servidor y calidad del stream influyen en la reproducción.

Legal y uso responsable

Usa XVB solo con fuentes autorizadas.

XVB es una herramienta neutral: no proporciona canales, credenciales ni contenido. Eres responsable de las fuentes que cargas. Usa XVB solo con streams y playlists autorizados (demos públicas, tus propios medios, proveedores IPTV con licencia o contenido que tengas derecho a distribuir).

Evita piratería, credenciales robadas, “free IPTV” o métodos para saltar paywalls, geo-restricciones o protecciones de copyright. Si no estás seguro, no uses la fuente.

Arquitectura del núcleo.

Totalmente client-side. No se necesita backend para reproducir. Basado en motores probados y APIs nativas del navegador.
