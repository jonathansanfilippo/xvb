¿Qué es XVB?

XVB (Extended Video Broadcasting) es un núcleo de reproducción web que se ejecuta íntegramente en el cliente. Está diseñado para gestionar IPTV y fuentes de transmisión directamente en el navegador, con especial énfasis en la fiabilidad, compatibilidad y rendimiento.

Este proyecto no es una "plataforma backend de IPTV" — es un motor de reproducción especializado. Tú aportas tus fuentes (playlists, flujos o archivos); XVB se encarga de la reproducción.

Por qué es importante.

  • Reproducción estable de larga duración (en vivo o VOD)
  • Inicialización y cambio de canales ultra rápidos
  • Compatible con los formatos de playlist IPTV comunes
  • Separación clara entre el motor de reproducción y la interfaz

Soporte de streaming.

Diseñado para entornos IPTV y una reproducción en vivo resiliente en redes inestables.

Protocolos y playlists

XVB soporta múltiples tecnologías de streaming utilizadas habitualmente:

  • HLS (HTTP Live Streaming, .m3u8)
  • MPEG-DASH (.mpd)
  • MPEG-TS flujos en directo (.ts)
  • M3U / M3U8 playlists
  • Flujos estándar MP4

Formatos de medios reproducibles

Según el soporte del navegador, XVB puede reproducir los siguientes formatos:

  • Vídeo: m3u8, mp4, webm, ogv, ts
  • Audio: mp3, aac, m4a, ogg, opus, wav, flac

La disponibilidad de los códecs puede variar según el navegador y el dispositivo.

Manual de Usuario

Comprendiendo la Interfaz

XVB cuenta con una barra de herramientas minimalista pero potente, diseñada para la velocidad y el monitoreo en tiempo real. Aquí tienes el detalle de cada elemento de tu tablero.

Métricas Live

  • Contador de Canales: Muestra el número total de flujos reproducibles cargados actualmente desde tus listas activas.
  • Estado EPG: Marca de tiempo en tiempo real de la última actualización de la Guía Electrónica de Programas (fuentes de Italia y Reino Unido).
  • Usuarios Online: Indica cuántos usuarios están conectados actualmente al ecosistema XVB en todo el mundo.

Navegación

  • Búsqueda Inteligente: Filtrado instantáneo. Empieza a escribir el nombre de un canal para encontrarlo inmediatamente sin desplazarte.
  • Nube de Muestras: Si no hay listas, haz clic aquí para cargar una lista técnica de prueba (MP4, AAC, MP3, OGG, MPD, TS y HLS).

Gestión de Listas

  • Añadir URL Remota: Importa enlaces M3U directos. Nota: este método se enfoca en enlaces estáticos directos para un rendimiento máximo.
  • Importación Local: Explora tu dispositivo para cargar archivos .m3u o .m3u8 guardados localmente.
  • Limpiar Caché: Borra al instante todas las listas almacenadas y los datos de la caché local del navegador.

Consejo Pro: Usa el botón Limpiar Caché si experimentas problemas tras actualizar un archivo local. XVB guarda los datos en el cliente para un inicio ultra rápido.

Playlists

Crea tu propia playlist M3U.

XVB carga playlists desde una URL o un archivo local. El formato estándar es M3U / M3U8, con metadatos (nombre, logo, grupo) seguidos de la URL del flujo.

Consejo: utiliza group-title para las categorías y tvg-logo para una interfaz más visual.

Ejemplo M3U

Guarda como mi-playlist.m3u (UTF-8). 

#EXTM3U

#EXTINF:-1 tvg-name="Canal Demo 1" tvg-logo="https://example.com/logo1.png" tvg-id="demo.canal.1" group-title="Noticias",Canal Demo 1
https://example.com/stream/canal1.m3u8

#EXTINF:-1 tvg-name="Radio Demo" tvg-logo="https://example.com/radio.png" tvg-id="demo.radio" group-title="Radio",Radio Demo
https://example.com/audio/radio.mp3

#EXTINF:-1 tvg-name="VOD Demo" tvg-logo="https://example.com/vod.png" group-title="Bajo demanda",VOD Demo
https://example.com/video/demo.mp4

Notas técnicas

Limitaciones del navegador.

XVB está sujeto a las reglas de seguridad web estándar: CORS (los servidores deben autorizar las peticiones), HTTP/HTTPS (los flujos no seguros se bloquean en sitios HTTPS) y la compatibilidad de códecs.

Uso responsable

Usa XVB con fuentes autorizadas.

XVB es una herramienta neutra: no proporciona contenido. Eres responsable de las fuentes cargadas. Evita la piratería y utiliza únicamente flujos de los que tengas derechos de acceso.

Arquitectura.

Integramente en el cliente. Basado en HLS.js, DASH.js y las API nativas de HTML5.
